Qu'est-ce que c'est ?

Ce plugin remplace l’affichage des images intermédiaire de Coppermine avec un système d’affichage en AJAX qui est largement configurable. Il fournit l'un des moyens les plus rapides possible afin de parcourir une galerie de photos.

Visibilité

Ce plugin fonctionne très bien graphiquement avec n'importe quel navigateur web moderne afin que vous ne perdiez pas de visiteurs en raison de problèmes de compatibilité. Les utilisateurs avec un navigateur sans JavaScript ou qui ont désactivé les scripts dans leur navigateur peuvent utiliser la galerie comme si le plugin Agrandissement ! (EnlargeIt!) N'était pas du tout là, donc ce plugin fonctionne bien si quelque chose ne va pas.

Les robots des moteurs de recherche ne seront pas concernés par l'indexation de votre galerie, ainsi ce plugin n'a pas un impact négatif sur le classement via votre moteur de recherche.

Caractéristiques

  • Boutons pour avoir une information sur image, image en taille réelle, photos préférées, téléchargement, histogramme
  • Navigation et avec boutons de fermeture
  • Navigation avec les touches fléchées ou la molette de la souris (molette de la souris en charge uniquement avec mozilla et IE)
  • Cinq types différents d'animation
  • Choisi la vitesse d'animation et des étapes
  • Choisissez la taille de la bordure, la couleur et l'ombrage
  • Centrage et l'assombrissement de l’écran (comme lightbox)
  • Agrandir la taille intermédiaire ou des photos en taille réelle
  • Ne pas exploser votre taille de page - seulement 22 ko de JavaScript et 1 ko de CSS est ajouté
  • Toujours trop grand pour vous ? Aller à http://enlargeit.timos-welt.de/english/11/building_blocks.php et générer vos propres petits, version personnalisée de la partie JavaScript
  • Support pour la lecture des fichiers Flash (*.swf)
  • Support pour la lecture des fichiers *.flv - il suffit de les télécharger dans votre CPG et ils fonctionneront
  • Support pour la lecture des fichiers *.divx via DivX Web Player (http://labs.divx.com/WebPlayer) - il suffit de télécharger un fichier *.divx dans votre galerie, et le plugin Agrandissement ! (EnlargeIt!) Le fera fonctionner sur Windows et Mac
  • Vous pouvez configurer pratiquement tout dans la page de configuration - mais ce n'est pas obligatoire

Comment faire l’installation

Ce plugin s’installe comme n'importe quel autre plugin, il n'y a pas étapes d'installation supplémentaires, de sorte que vous devriez bêtement faire comme suggéré dans la section correspondante de la documentation qui vient avec Coppermine.

Mise à jour

Si vous mettez à jour ce plugin à partir d'une version précédente, veillez à toujours désinstaller complètement l'ancienne version avant d'installer la nouvelle, et à vider le cache de votre navigateur après la désinstallation.

Comment faire pour la désinstallation

Désinstaller via gestionnaire de plugins.

Configuration

Utilisez le bouton supplémentaire à l'intérieur du menu administrateur de Coppermine ou dans la page du gestionnaire de plugins pour accéder à l'écran de configuration du plugin Agrandissement ! (EnlargeIt).

Type d'élargissement

Activer pour

Cases possibles:
  • Administrateurs
  • Utilisateurs enregistrés
  • Invité
Cette option détermine qui va voir le plugin Agrandissement ! (EnlargeIt) en premier lieu. Notez que cela ne change rien concernant, quoi que ce soit sur les autorisations d'afficher ou de télécharger dans Coppermine: ces paramètres sont encore contrôlés par les mécanismes « normaux » de Coppermine. Au lieu de cela, cette option détermine pour laquelle le groupe du plugin Agrandissement ! (enlargeIt) est activé en premier lieu. Tous les visiteurs qui n'appartiennent pas à un groupe activé du plugin Agrandissement ! (enlargeIt) verront le mécanisme classique de Coppermine pour afficher des images, c'est à dire qu'ils seront envoyés à partir de l'écran des miniatures, à l'écran d'image de taille intermédiaire (displayimage.php).
Par défaut, les trois cases sont cochées.

Zoom de l’image

Précise le "facteur de grossissement" d’une vignette accédé (en cliquant dessus). Il y a trois options possibles:
  • Taille intermédiaire
  • Taille réelle
  • Forcer la taille intermediaire

Maximiser la méthode

Détermine comment les images en taille réelle doivent être ouvertes.
Available options:
  • Dans une fenêtre pop-up (non recommandée)
  • Ouvrir en tant que snippet AJAX (recommandée)
Cette option s'applique uniquement si vous permettez les images en taille réelle, en premier lieu, afin que les deux l'option de Agrandissement (EnlargeIt) "Zoom de l’image" ainsi que les options de configuration de Coppermine qui ont un impact sur le redimensionnement des images en taille réelle puissent être prises en compte.

Animation

Type d'animation

Préciser l'animation qui doit être utilisée quand un utilisateur clique sur la vignette ou lorsqu'on passe d'une image à l'autre.
Les options possibles:
  • Aucun
    Pas d'animation du tout - la case Agrandissement (EnlargeIt) apparaitra instantanément sans aucun effet d'animation.
  • Apparition et disparition graduellement du fondu
  • Glissement
  • Glissement par à coup
  • Glissement doux
  • Glissement rapide
  • Glissement par le haut
  • Glissement par la gauche
  • Glissement par le haut gauche

Temps entre les étapes d'animations

L'animation est en fait un ensemble d'étapes d'animation individuels qui se déclenche quand un visiteur accède la vignette. Considérez-les comme les images d'une vidéo. Vous pouvez spécifier le délai en millisecondes entre les étapes d'animations, cette option détermine la vitesse de l'animation. La valeur par défaut est de 12 ms. Vous pouvez jouer avec ce paramètre pour le type d'animation que vous souhaitez voir.

Nombre d'étapes pour les animations

Le nombre d'étapes d'animation. La valeur par défaut est 18. La valeur minimale est de 4, la valeur maximale est de 30. Ce paramètre détermine la façon dont l'animation sera vue. Plus la valeur est grande, plus lisse sera l'animation. L'inconvénient de cette option à une valeur élevée, est qu'elle provoque des décalages sur les ordinateurs lents des visiteurs, les plus âgés. Lors de l'essai de ce paramètre, ne vous ne laissez pas bercer d’illusions en croyant que tout ira bien si cela semble OK sur votre PC: sur d'autres machines moins puissantes un grand nombre d'étapes d'animation peut causer des problèmes.

Effet de transparence pour les animations glissées

Lorsque cette option est cochée, le script tente d'utiliser la transparence pour l'effet de glissement, ce qui aura un impact sur les performances des ordinateurs les plus âgés. Le réglage par défaut est désactivé.

Bordure

Paramètres concernant la bordure autour des images individuelles lors de l'utilisation d’Agrandissement (EnlargeIt) pour afficher une image qui réside dans votre galerie.

Largeur de la bordure

La largeur de la bordure est en pixels. La valeur par défaut est 22. Mettez à zéro pour désactiver la bordure. Tous les paramètres ci-dessous concernant la bordure ne seront pas pris en compte si vous définissez la largeur de la bordure à 0.

Couleur de la bordure

Couleur de fond de la bordure, en utilisant la notation hexadécimale avec le premier caractère #. Vous pouvez entrer la valeur manuellement ou utiliser le sélecteur de couleur..
La valeur par défaut est # FFFFFF, ce qui est blanc uni.

Texture de la bordure

En plus de la couleur de fond, vous pouvez affecter une texture de fond de la bordure. Lorsque vous spécifiez une texture, la couleur de fond ne sera pas prise en compte. Le réglage de la texture des bordures est une liste déroulante où vous pouvez sélectionner l'une des images. Un aperçu sera affiché dans la colonne à droite. La valeur par défaut est "Aucune ".
Les options disponibles sont:
  • Aucune
  • Marbre
  • Métal brossé
  • Blanc métal
  • Blanc métal 2
  • Bleu métal
  • Rouge métal
  • Vert métal
  • Argent métal
  • Noir métal
  • Pluie
  • Petite pluie
  • Bois clair
  • Bois foncé
  • Papier
  • Cuir
  • Vert foncé
  • Vert liquide
  • Chocolat

Bords arrondis

Case à cocher pour permettre les bords arrondis. La fonction bordure arrondie ne fonctionne qu’avec Mozilla / Safari seulement, de sorte qu'elle ne fonctionne pas avec IE. Toutefois, si vous l'activez, rien de mauvais ne se passera pour les visiteurs qui ont un navigateur qui ne prend pas en charge cette fonctionnalité: il n'y aura aucun message d'erreur si vous activez cette fonctionnalité. Au lieu de cela, l'objet Agrandissement (EnlargeIt) sera affiché avec des bords "ordinaire", non arrondis.
Est désactivée par défaut.

Ombre

Taille de l'ombre

Ce paramètre vous permet de spécifier la largeur de l'effet d'ombre. L'effet d'ombre s'étire à la fois vers la droite et vers le bas, comme si la lumière venait du haut à gauche de l'écran. Réglez cette option à zéro pour désactiver l'effet d'ombre entièrement. Les valeurs peuvent aller de 1 (petite ombre) à 9 (grande ombre).

Opacité de l'ombre

Déterminer comment l'ombre sera "sombre ", c'est à dire l’opacité de celle-ci. La valeur peut être comprise entre 1 et 30. Le réglage par défaut est 20.

Couleur de l'ombre

Déterminer la couleur de l'ombre - peut être utile pour des thèmes sombres. Précisez la couleur en utilisant la représentation hexadécimale. Utilisez le sélecteur de couleur pour choisir très facilement la couleur avec votre souris.
La valeur par défaut est #000000, c'est à dire noir.

Barre de titre

Voir la barre de titre lorsque les boutons sont inactifs

Je n'ai aucune idée de ce que fait cette option. Si d’autres développeurs pouvaient préciser s’il vous plaît ? Merci. Joachim
Par défaut cette option est activée.

Couleur du texte de la barre de titre

La couleur de police pour la barre de titre, c'est à dire la couleur de la représentation en hexadécimal avec le signe # pour, le titre de l'image individuelle. Il est nécessaire d'utiliser une couleur ayant un bon contraste par rapport à l'arrière plan de la barre de titre, à savoir la couleur de la bordure ou de la texture des bordures. Utilisez le sélecteur de couleur pour définir la couleur en utilisant votre souris.
La valeur par défaut est #44554, ce qui est d'un vert foncé.

La couleur de l'arrière plan de la zone AJAX

La couleur de l’arrière plan pour le contenu d’Agrandissement (EnlargeIt). Utilisez le sélecteur de couleur pour définir la couleur en utilisant votre souris.
La valeur par défaut est #666677, qui est un gris moyen.

Action

Centrage de l'image agrandie

Détermine si l'image doit être centrée à l'intérieur de l’objet Agrandissement (EnlargeIt!). Est par défaut (et recommandé) sur "on".

Autoriser images agrandies à être déplacés

Lorsqu'il est activé, le visiteur peut déplacer l'objet Agrandissement (EnlargeIt) autour de l'écran en utilisant sa souris. Ceci est souvent utile si le thème est étroit et qu’il y a des informations sous l'image agrandie que le visiteur souhaite voir.
Est désactivée par défaut. Il est recommandé de le mettre sur on.

Navigation avec la molette de la souris

Lorsque cette option est activée les visiteurs peuvent utiliser la molette de leur souris pour faire défiler les images au lieu d'avoir à cliquer sur les touches PRÉCÉDENTE / SUIVANTE, qui rende la navigation à l'écran comme un jeu d'enfant. Le seul inconvénient de cette option réside dans le fait que le comportement "classique" avec la molette de la souris (défilement de la fenêtre du navigateur) est désactivé en faveur de la navigation via la molette de la souris.
Par défaut, la navigation via la molette de la souris est activée.

Assombrir l’écran lors des agrandissements

Je n'ai aucune idée de ce que fait cette option. Si d’autres développeurs pouvaient préciser s’il vous plaît ? Merci. Joachim
Les options possibles sont:
  • Non
  • Oui: Assombrir lorsqu'une image est montrée
    Option par défaut
  • Oui: reste sombre lorsque vous utilisez la navigation

Forçage de l’assombrissement

Déterminer combien l'écran autour de l'objet Agrandissement (EnlargeIt) est sombre, à savoir l’assombrissement de l'effet boite grise. Les valeurs possibles vont de 0% à 100%.
La valeur par défaut est de 20%

Vitesse d’assombrissement

Déterminer la vitesse à laquelle l'effet d'assombrissement doit être: la plage de valeur possible est de 1 à 20, ou 1 signifie "assombrir immédiatement" et 20 signifie "assombrir très lentement".
La valeur par défaut est 20.

Cette option n'a de sens que si vous voulez que vos visiteurs d'accèdes à vos images via les forums et autres. Habituellement, c'est le cas si vous avez une galerie d'avatar ou similaires. L’option recommandée est "off" n’activez cette option que si vous comprenez l'impact de celle-ci.
Le tableau bbcode ressemble à ceci:

Est "désactivé" par défaut.

Affichage du bouton "Histogramme

Détermine si un bouton histogramme () doit être disponible sur l'interface de Agrandissement (EnlargeIt).

Ce plugin possède une option pour créer et afficher un histogramme pour toutes les images de votre galerie. Il est au-delà de la portée de cette documentation de fournir une explication détaillée sur ce que les histogrammes sont et ce à quoi ils sont supposés être utiliser.

Une explication sommaire: l’histogramme d'une image est une représentation graphique de la distribution des tons d'une image numérique.

Utilisation recommandée: vous ne devez fournir l'histogramme que si votre public cible sont des photographes et sachant ce qu'est un histogramme et ce pourquoi il peut être utilisé.

Liens utiles:

  • Wikipedia → Histogramme (imagerie numérique)
  • The Luminous Landscape → Tutorials → Understanding Histograms
  • Michael Mercadante → Understanding Histograms
A côté de la case à cocher qui vous permet de basculer l'utilisation histogramme sur On ou Off il y a la méthode que votre système utilise pour l’affichage concernant la version de la bibliothèque GD sur votre serveur. Comme la création d’histogramme demande au minimum GD2, vous ne pouvez pas activer la case à cocher si votre version de GD n'est pas suffisante. Pour plus de détails, connectez-vous sur votre galerie Coppermine en tant qu'administrateur et puis cliquez sur le bouton phpinfo dans le menu Information. Dans l’affichage de phpinfo, accédez à la section GD pour plus de détails. S'il vous plaît comprenez qu’il est au-delà de la portée du support de Coppermine de vous apprendre la configuration d’un serveur web, vous aurez donc besoin de comprendre cela avec votre hébergeur, si vous n'avez pas le support GD2 d’activé.
Les options qui correspondante à "Durée de vie du cache du fichier histogramme" sont grisé si le bouton histogramme n'est pas activé, car elles ne s'appliquent pas.
Il est important de comprendre l'impact que la création d'histogrammes peut avoir sur votre serveur: ils peuvent causer une augmentation de la puissance nécessaire de façon considérable, car la création d'histogrammes est un processus qui demande un fort besoin des ressources: l'image est chargé en mémoire et ensuite traitées via diverses procédures qui ne sont pas insignifiante.
Est "désactivé" par défaut.

Affichage des boutons de "Navigation"

Détermine si les boutons précédent () et suivant () seront affichés le cas échéant (c'est à dire s’il y a une image précédente / suivante dans un album) dans l'interface d’Agrandissement (EnlargeIt) pour faciliter la navigation d'une image à l'autre. Si cette option n'est pas activée, vos visiteurs seront obligés de fermer chaque image et d’aller à l'image suivante en cliquant sur la vignette correspondante. Si vous activez cette option, ils peuvent aller d'une image à l'autre sans avoir besoin de cliquez sur la vignette. Le seul inconvénient de permettre cette option est une interface un peu plus encombrée.
Est (et est recommandé) "activé" par défaut.

Détermine si un bouton de fermeture () sera affichée dans l'interface d’Agrandissement (EnlargeIt).
Est (et est recommandé) "activé" par défaut.

Multimédia

Formats d'image

Sélectionnez les formats de fichiers où le plugin Agrandissement (EnlargeIt!) doit être utilisé via la sélection de plusieurs champs. Tous les autres formats de fichiers (si présents dans votre galerie) seront affichés en utilisant le mécanisme "classique" de Coppermine. Les lignes marquées d'un astérisque (*) indiquent les formats de fichiers qui sont refusés via la configuration de Coppermine, donc il est fort probable que dès le départ ces fichiers n'existent pas dans votre galerie.

Ce paramètre ne détermine pas ce que vous pouvez télécharger comme fichiers – Pour le contrôle de cela, allez dans la configuration de Coppermine.

Les lignes marquées d'un astérisque (*) indiquent les formats de fichiers qui sont refusés via la configuration de Coppermine, donc il est fort probable que dès le départ ces fichiers n'existent pas dans votre galerie.

Configuration du plugin

Détermine si un lien vers le panneau de configuration pour le plugin Agrandissement (EnlargeIt!) doit résider dans le menu configuration. Une fois que vous avez correctement configuré le plugin Agrandissement (EnlargeIt) vous devez désactiver le lien du menu configuration pour faire apparaître le menu configuration moins encombré.
Est "activé" par défaut.

Durée de vie du cache du fichier histogramme

Par défaut les options qui correspondent à la durée de vie du cache du fichier histogramme sont grisées (désactivés), sauf si vous activez l'option Affichage du bouton "Histogramme".
Les options disponibles sont:
  • Illimitée
  • XX jours
  • Taille maximum total du fichier de XX MiB
Concept
Pour comprendre l'impact de ces paramètres, vous devez comprendre dans un premier temps ce qu’est le bouton histogramme: chaque fois qu'un visiteur accède à une image en utilisant l'interface d’Agrandissement (EnlargeIt) et qu’il clique le bouton "histogramme", le plugin vérifie si un fichier histogramme existe déjà. S’il n'existe pas, le script crée un nouveau fichier d’histogramme et l’enregistre. Ce fichier enregistré est par la suite intégrés dans les fichiers d’Agrandissement (EnlargeIt), de sorte que le visiteur peut voir l'histogramme actuel.
Permissions

Le script tente de créer le fichier de l'histogramme dans le même dossier où l'image originale réside. Par la suite, si les autorisations au niveau du système de fichiers ne sont pas configurés correctement (rappelez-vous: Coppermine a besoin des autorisations RXW sur le dossier albums et tout ce qu'il contient), le plugin ne sera pas en mesure d'enregistrer le fichier de l’histogramme. Par la suite, si le fichier de l’histogramme ne peut être créé, vous devrez d'abord vérifier si le script a obtenu les autorisations nécessaires pour créer le fichier en premier lieu. Reportez-vous à la documentation fournie avec Coppermine, la section "Configuration des permissions" pour en savoir plus.

Schéma de nommage
Le schéma de nommage pour le fichier histogramme est assez simple: il est similaire à celui pour les images avec le préfixe thumb_ le fichier d’histogramme lui, obtient le préfixe histogram_. Par la suite, vous devrez vous assurer de ne pas télécharger des fichiers en utilisant le téléchargement par HTTP classique qui sont préfixés de cette façon, afin de s'assurer qu'il n'y a pas de nommage identique (chose qui devrait être rarement le cas de toute façon).
Si votre fichier d'origine réside à:
http://votresite.fr/votre_dossier_coppermine/albums/userpics/10001/votreimage.jpg et votre vignette sera par la suite à
http://votresite.fr/votre_dossier_coppermine/albums/userpics/10001/thumb_votreimage.jpg, Le fichier sera de l’histogramme sera à
http://votresite.fr/votre_dossier_coppermine/albums/userpics/10001/histogram_votreimage.png
Taille du fichier
Les dimensions et la taille du fichier de l'histogramme ne correspondent pas aux dimensions ou à la taille de votre image originelle, vous n'avez donc pas à être inquiet sur une possible trop grande consommation d'espace web; ci-dessous il y a un tableau qui indique des exemples sur la consommation d'espace web de certaines images:
# Fichier originel Fichier d'histogramme
Type de fichier Taille du fichier Dimension de l'image Type de fichier Taille du fichier Dimension de l'image
1 jpg 32.69 KiB 391 x 600 png 1.64 KiB 284 x 164
2 jpg 20.89 KiB 385 x 415 png 2.10 KiB 284 x 164
3 jpg 334.30 KiB 1437 x 1919 png 1.22 KiB 284 x 164
4 png 304.30 KiB 600 x 450 png 0.23 KiB 284 x 164
5 jpg 321.20 KiB 2048 x 1536 png 1.34 KiB 284 x 164
Comme vous pouvez le voir, les dimensions du fichier histogramme sont toujours les mêmes: leur résolution 284 x 164 pixels est codé en dur dans le plugin Agrandissement (EnlargeIt). De plus le format du fichier de l'histogramme est toujours png.
Suppression automatique du cache du fichier de l’histogramme après un certain temps
L'option "Durée de vie du cache du fichier histogramme → XX jours vous permet de spécifier la suppression automatique du cache de l’histogramme après un nombre de jours déterminé. De cette façon, les fichiers d’histogramme qui sont plus âgés que le nombre de jours que vous indiquez seront supprimés chaque fois qu'un visiteur clique sur n'importe quel bouton d’histogramme dans la galerie. Vous pouvez spécifier un nombre de jours compris entre 1 et 365.
La valeur par défaut est de 30 jours. Ne s'applique que si le bouton radio en face de cette option a été coché en premier lieu.
Suppression automatique du cache de l’histogramme via la taille maximum du fichier
L'option "Durée de vie du cache du fichier histogramme → Taille maximum total du fichier de XX MiB vous permet de spécifier un maximum concernant la consommation d'espace web de l’ensemble des caches d’histogramme en mémoire. Si le total de la taille de tous les fichiers d’histogramme est plus grand que la valeur que vous indiquez ici, les fichiers d’histogrammes les plus anciens seront supprimés jusqu'à ce que le total soit inférieur à la valeur que vous avez défini. Cette option peut être utilisée pour s'assurer que la consommation d'espace web ne soit jamais plus grande que votre espace web maximum. La routine de collecte des caches qui va supprimer les fichiers les plus anciens sera déclenchée à chaque fois qu'un visiteur cliquera sur n’importe quel bouton histogramme. Vous pouvez spécifier un nombre de jours compris entre 1 et 99.
La valeur par défaut est de 10 Mio. Ne s'applique que si le bouton radio en face de cette option a été coché en premier lieu.
Informations supplémentaires concernant le cache
Si le cache de l’histogramme n'est pas vide (c'est à dire si l'option "Affichage du bouton "Histogramme"" a été activé et que le formulaire à été valider, il n'y aura plus d’informations disponibles concernant l’option: "Durée de vie du cache du fichier histogramme" la taille réelle du cache en mémoire (le nombre de fichiers ainsi que la consommation d'espace web) sera affiché.
Valeur par défaut
La sélection par défaut pour la mise en cache de l’histogramme est "illimitée", ce qui signifie que le cache ne sera jamais effacé. Recommandé si vous avez suffisamment d'espace web disponible.

Nuanciers

Dans le panneau de configuration du plugin Agrandissement (EnlargeIt!) il y a plusieurs sélecteurs de couleurs qui vous permettent de spécifier une couleur en cliquant dessus. Pour utiliser le sélecteur de couleur dont vous avez besoin d'élargissez celui-ci dans un premier temps (par défaut, il est réduit) en utilisant les liens sélecteur de couleur. Vous pouvez alors sélectionner une couleur dans le cercle et ainsi la nuance à l’intérieure de la case. En cliquant sur la case du champ de couleur actuelle. Tous les champs de couleur acceptent la notation de couleur en hexadécimal avec le signe dièse devant.

Sélection de plusieurs champs

Maintenez la touche [Ctrl] pour sélectionner plusieurs lignes sur un champ à sélection multiple. Pour ne choisir aucune ligne ou tous, désélectionner la dernière ligne sélectionnés en maintenant la touche [Ctrl] enfoncée, puis en cliquant sur cette dernière ligne restante.

Premiers secours - si elle ne fonctionne pas

  • Vérifiez si le visiteur est autorisé à accéder à enlargeit.js et le reste des dossiers et fichiers du plugin (http://chemin.d'accès.à.votre.galerie/plugins/enlargeit/js/enlargeit.js).
    Le dossier complet doit être accessible via le web pour les visiteurs, car il contient le JavaScript et les fichiers graphiques.
  • Si vous utilisez le plugin SEF, n'oubliez pas de changer le réglage dans la page de configuration d’agrandissement (EnlargeIt!).
  • Si vous utilisez un thème très particulier ou un thème qui n’est pas valide HTML / XHTML, ce plugin ne fonctionnera probablement pas. Je peux vivre avec cela, et il le faut !
