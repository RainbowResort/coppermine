Plugin for CPG 1.5.x: FetchContent

What's this?

Display content taken from coppermine on non-coppermine-driven pages. Might later become the successor of cpmFetch.

Minimum requirements

-

Install and uninstall

Install/uninstall this plugin as any other plugin, using the plugin manager. There are no particular install/uninstall steps that differ from "regular" plugins.

Configuration

After installing the plugin you can access the plugin's configuration screen at any time by clicking on the corresponding link on the plugin manager page.
On the plugin configuration screen you can enable or disable how the plugin works.

Individual file (image) settings

The following options apply for individual images / files fetched by the script.

Deny access action

// Specifiy what should happen if the visitor is denied to see the image
// Possible options: 0 - output nothing at all, resulting in a broken image
//                   1 - Display a blank, 1x1 pixel image (recommended, default)
//                   2 - Display a placeholder image that tells the visitor
//                       about being denied to view the image he requested
//                   3 - try to display the image no matter what, ignoring 
//                       permissions (troubleshooting only!)

Check referer

// Specify if the referer should be checked (not implemented yet)
// Possible options: 0 - don't check
//                   1 - only allow the domain that the gallery resides on
//                   string - list of allowed domains, separated with a pipe (|)

Enable logging

// Specify if the actions of this plugin should be logged
// Possible options: 0 - don't log
//                   1 - only log errors
//                   2 - log everything

Allow other files than images

// Specify if non-images should be allowed to be fetched as well
// Possible options: 0 - don't allow non-images
//                   1 - allowed, but only if URL parameter "filetype" is explicitely set to
//                       'document', 'audio', 'movie' or 'any'
//                   2 - allowed, but only if URL parameter "imageonly" is NOT set
//                   3 - allowed without further checking

Fetchcontent debugging

// Set to 1 during debugging (i.e. if you're a coder and you're developing this plugin).

//                   1 - debugging on: there will be no header redirection. 
//                       Instead, there will be output echoed.
