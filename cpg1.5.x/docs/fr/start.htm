Configuration Initiale

Il y a certains paramètres de la configuration de Coppermine que vous devrez modifier tout de suite après l'installation.

Identifiez vous avec l'identifiant et le mot de passe administrateur que vous avez défini lors de l'installation, cliquez sur le lien "Montrer les controles administrateurs" si il est visible, allez à la page de Configuration et commencez à configurer votre galerie. Notez que même si vous êtes membre du groupe administrateur, vous devrez activer la fonction "Montrer les controles administrateurs". Dans les versions précédentes, ce lien était nommé "mode administrateur", mais il a été renommé parceque certains pensaient que le fait de passer du mode administrateur au mode utilisateur leur montrait ce que les visiteurs pouvaient voir (ce qui n'est pas le cas).

  • Courriel de l'administrateur de la Galerie
  • URL du dossier de la Galerie Coppermine

Il y a certains paramètres de la configuration qui ne peuvent pasêtre changés plus tard (si il y a déjà des fichiers dans la base de donnée) - assurez vous de les configurer correctement dès le départ (ceux marqués par une astérisque "*") .

Certains paramétrages ont de sérieux impacts sur les performance et doivent être regardés attentivement, en particulier si vous êtes hébergés sur un hébergeur mutualisé ou gratuit.

Structure des catégories, albums et fichiers

Coppermine Photo Gallery fonctionne de la manière suivante:

  • Les fichiers sont placés dans des albums
  • Les albuls sont organisés en catégories
  • Les catégories peuvent être imbriquées (en sous-catégories)

Si vous n'avez pas l'intention d'avoirbeaucoup d'albums, vous ne souhaiterez peut être pas utiliser les catégories. Si c'est le cas, ne créez simplement pas de catégories et tous vos albums apparaions sur la page principale de votre galerie.

Il y a toutefois une catégorie spéciale nommée "User galleries/Galerie Utilisateurs". Cette catégorie ne peut pas être effacée. Si un utilisateur appartient à un groupe qui "peut avoir une galerie personnelle" et que ce paramètre est défini à OUI, il aura le droit de crée ses propres albumset sa galerie sera unesous-catégorie "User galleries". Ce lien n'est toutefois pas visible pour les visiteurs de votre sitesi vous n'autorisez pas vos utilisateurs à télécharger des images et à avoir leurs propres albums.

L'administrateur peut crée des albums dans n'importe quelle catégorie. Un utilisateur non-administrateur ne peut crée des albums que dans "User galleries/son_nom_d_utilisateur".

Vous pouvez toutefois renommer "User Galleries". Pour renommer la catégorie "User galleries" et sa description, allez simplement dans votre gestionnaire de catégories et changez le nom (par exemple en traduisant les mots "User galleries" dans votre langue).

Images redimensionnées

Un des avantages proncipaux de Coppermine est le fait que le script crée les vignettes et les images intermédiaires pour les images originales téléchargées. Celà se fait une fois (au téléchargement), néanmoins le processus de création de images redimenssionnées peut être relancé en utilisant l'outil d'administration. Les copies redimensionnée de vos images originales sont stockées dans le même répertoire que vos originaux - elles diffèrent uniquement au niveau des préfixes.

Les images redimensionnées suivantes sont générées lorsque vous téléchargez une image:

  • Vignettes

    Les vignettes sont crées en fonction des otpions définies dans les "paramètres des vignettes" dans la configuration, si le fichier téléchargé est plus grand en terme de dimensions que les dimensions définies dans le paramétrage des vignettes. Les vignettes sont essentiellement utilisées sur la page de vignettes des albums, mais aussi à d'autres endroits (par exemple dans la page de profil ou comme vignette des albums ou des catégories). La création de vignettes augmente grandement la vitesse de chargement de votre galerie - si Coppermine ne générait pas de vignettes et affichait à la place les images originales (en utilisant les atributs HTML pour les afficher en plus petit), la page des vignettes serait terriblement longue à charger.
    Pour les dimensions de vos vignettes, vous devez trouver le bon compromis: des vignettes avec de petites dimensions génèrent de petits fichiers et donc la page se charge très rapidement. L'inconvénient de petites dimensions sont manifestes: si vous entrez des valeurs trop petites, vos visiteurs ne se rendront pas compte de ce qu'il y a sur la photo et ne cliqueront pas sur la vignette pour voir l'image originale. Il n'y a pas de paramètres idéaux pour la taille des vignettes, celà dépends beaucoup du sujets de votre galerie, Toutefois nous vous conseillons d'utiliser en règle générale entre 50 et 250 pixels .
    Si vous téléchargez un fichier plus petit en terme de dimensions que la largeur/hauteur paramétrée pour les vignettes dans la configuration de Coppermine, la création de la vignette sera ignorée et l'image originale sera utilisée comme vignette.

  • Images intermédiaires

    L'image intermédiaire est crée si vous n'avez pas désactivé l'option "Crére une image intermédiaire" ou si l'image téléchargée est plus petite ou égale en terme de dimensions au valeurs pour largeur/hauteur que vous avez défini pour Largeur ou hauteur Max d'une image intermédiaire. es images intermédiares sont essentiellement utilisées pour l'affichage de la page des images intermédiaires (displayimage.php) que l'utilisateur voit une fois qu'il a cliqué sur la vignette. La page des images intermédiares peut contenir toutes sortes d'informations complémentaires sur l'image (comme le nom du téléchargeur, le titre, la description, des mots clés etc...) et a un controle de navigation permettant aux visiteurs d'aller directement à l'image suivante/précédente d'un album sans être obligé de retourner à la page des vignettes. En plus, la page des images intermédiares comprends des sections pour une interaction avec les utilisateurs (comme le vote ou les commentaires, si vous les avez autorisés dans la configuration).
    Le plus gros avantage d'avoir des images intermédiaire réside dans le fait que celle ci est prévue pour s'addapter à l'écran du visiteur, quelque soit sa résolution. Il en résulte une navigation transparente pour vos visiteurs. Il est préférable d'utiliser des valeurs "sures" pour Largeur ou hauteur Max pour les images intermédiaires pour être sur qu'elle tienne à l'écrant(y compris avec les contrôles supplémentaires de la page). Comme l'espace necessaire à ces contrôles dépend du theme utilisé et que vous ne pouvez par être sur de la résolution d'écran de vos visiteurs, il est préférables d'utiliser des dimensions pour les images intermédiares se situant entre 400 et 800 pixels.
    La création des images intermédiaires accélère aussi le chargement des pages, particulièrement pour les utilisateurs avec une connection lente, mais elles prennent bien entendu de la place sur le serveur d'hébergement. Ne désactivez la génération d'images intermédiaires que si vous n'avez pas beaucoup d'espace sur votre serveur. Les images intermédiares sont stockées dans le même répertoire que les images originales et ont le préfixe "normal_".
    Admettons que vous ayez paramétré les dimensions maximales pour les images intermédiares à 400 pixels. Vous (ou un de vos utilisateurs à qui vous avez autorisé le téléchargement) téléchargez une image nommée "monimage.jpg" qui a 800 pixels de large et 600 pixels de haut. Une image intermédiare sera crée lors u processus de téléchargement, et sera nommée "normal_monimage.jpg" avec une résolution de 400 x 300. Une autre image est alors téléchargée avec pour nom "foobar.jpg" qui a 1024 par 768 pixels: là aussi, une image intermédiaire sera crée avec pour nom "normal_foobar.jpg" et qui sera aussi de dimensions 400 par 300 pixels. Une troisème image est téléchargée nommée "grosseimage.jpg" avec pour dimensions 1280 par 800 pixels. L'image intermédiaire crée ("normal_grosseimage.jpg") aura 400 par 250 pixels. Vous noterez que les dimensions de la troisième image intermédiaire est différente des deux autres. La raison est que la création des images intermédiaires respecte le ration des images originales. Néanmoins, vos visiteurs, lorsqu'ils parcoureront l'album qui contient les trois images, verra toujours une image inclue dans la page et qui aura 400 pixels de large, de manière à ce que la page soit toujours visible sans barre de défilement horizontale qui forcerait le visiteur à faire défiler les pages sans arrêt.

  • Images Originales

    Les images originales sont affichées dans une fenêtre pop-up lorsque l'utilisateur clique sur l'image intermédiaire si l'image originale est plus grande en terme de dimensions que l'image intermédiaire.
    Si vous avez paramétré les dimensions maximales pour les images intermédiaire à 400 pixels et que vous autorisez le téléchargement de plus grandes images. Deux fichiers sont téléchargés: le fichier "premiere_image.jpg" qui a 800 par 600 pixels, le ficher "seconde_image.jpg" qui a 400 par 300 pixels. Si un visiteur regarde l'image intermédiaire affichée pour "premier_image.jpg", il/elle pourra cliquer sur l'image intermédiaire et l'image originale s'afichera dans une fenêtre pop-up. Si il/elle regarde l'image intermédiaire affichée pour "seconde_image.jpg", le fait de cliquer sur cette image intermédiaire n'aura aucun effet, puisque visulaiser l'image originale dans une fenêtre pop-up n'a pas vraiment de sens: l'image intermédiaire affiche déjà l'image dans la résolution la plus grande disponible.
    Vous pouvez demander à Coppermine de ne jamais redimenssionner les images originales qui sont téléchargées et de cette manière (en théorie toutefois) de préserver l'image originale sans modifications, il y a toutefois de gros inconvénients: avec les appareils photos modernes capables de faire des images plus grandesen terme de dimensions que les capacités des écrans de vos visiteurs, l'image originale affichée dans une fenêtre pop-up ne s'adaptera jamais à la taille de l'écran de vos visiteurs, les forçant à utiliser les barres de défilement sans arret. En plus, l=d'énormes images utiliseront beaucoup d'espace web et se chargeront très lentement pour les visiteurs ayant des connections avec peu de bande passante.
    C'est pourquoi Coppermine contient une option pour redimensionner les images originales au moment du téléchargement à une réolution maximale - il est fortement recommandé d'utiliser cette option aussi bien pour les téléchargements de l'administrateur que pour ceux des visiteurs réguliers. Cette option se trouve dans la configuration de Coppermine et s'appelle Redimensionner automatiquement les images qui dépassent la hauteur et/ou la largeur maximale . Il est recommandé d'utiliser des valeurs comprises entre 800 et 1200 pixels. dans cette documentation, nous nous référons à ces images comme étant "originales" bien qu'elles aient été initialement redimensionnées par Coppermine pendant le téléchargement. Techniquement parlant, ce ne sont plus des images originales, mais des copies redimensionnées, mais comme cette option a été ajoutée après la publication de la première version de Coppermine, la documentation se réfère à ces fichiers comme "originaux". Les images originales sont stockées sans préfixe, utilisant le nom de fichier donnée par le téléchargeur. Le nom de fichier est uniquement modifié si il contient certains caractères interdits ou spéciaux, spécifiés dans les options de configuration de Coppermine Caractères interdits pour les noms de fichiers.

Coppermine utilise des librairies graphiques (soit GD ou ImageMagick) qui doivent exister sur votre serveur et capable de créer des images redimensionnées (regardez les "minimum requis"). Ces librairies ne sont pas en mesure de redimensionner les vidéons ni de préserver les méta données comme les données EXIF ou IPTC. C'est la raison pour laquelle les vignettes et les images intermédiaires ne peuvent être crées que pour le type de fichiers images supportés par la librairie que vous utilisez.

Il peut y avoir des problèmes lors du redimensionnement de grandes images, parce que le redimensionnement utilise beaucoup de ressources serveur. Référez vous à la section "Résolution de problèmes de téléchargement" de la documentation si vous avez des problèmes avec de gros fichiers (ou pour faire fonctionner le téléchargement au départ).

Paramètres recommandés pour le redimensionnement

Vous devez comprendre que le fait de changer le paramétrage de redimensionnement n'aura un impatc que sur les images téléchrgées après la modification. Les images déjà présentes au moment des changements que vous ferez dans cette partie de la configuration ne seont pas automatiquement redimensionnées suivant les nouveaux paramètres - vous devrez avoir recours au procesus de redimensionnement en utilisant l'option correspondante "mise à jour des vignettes et/ou des images redimenionnées" des outils administrateur et passer dans tous les albums pour utiliser ces nouveaux paramètres.

Pas d'images de grande taille dans une fenêtre pop-up

Si vous ne projetez pas du tout d'utiliser l'affichage des images de grande taille dans une fenêtre pop-up, par exemple si les images doivent toujours être affichées à l'intérieur de votre galerie Coppermine, vous pouvez paramétrer coppermine de cette manière afin d'économiser de l'espace sur le serveur.

Vous devez comprendre que vous devrez satisfaire vos avec une plus petite résolution. Si vous pensez que vos visiteurs ont une résolution de 1024 x 768 pixels ou mieux, vous devriez paraméter la taille des images embarquées à 800, parceque vous aurez besoin d'espace complémentaire pour l'affichage des espaces et des autres controles de la page.

Admettons par exemple que la taille maximale que vos visiteurs sont en mesure de voir soit800 pixels.

Allez dans la configuration de Coppermine et paramétrez:

  • "Créer des images intermédiaires" sur "non"
    De cette manière, les images originales seront utilisées dans la page displayimage (affichage après le clic sur la vignette)
  • "Largeur ou hauteur Maximale des images/vidéos" à "800"
    Cela prendra en compte la résolution que vous voulez avoir pour les images affichées dans la page displayimage
  • "Largeur ou hauteur maximale pour les images/vidéos téléchargées" à "800"
    Il en résultera un message d'erreur si l'utilisateur essaye de télécharger des images avec une plus grande résolution si vous paramétrez l'option suivante "Redimensionner automatiquement les images qui sont plus grandes que la largeur ou hauteur maximale" sur "Non". Toutefois si vous réglez, "Redimensionner automatiquement les images qui sont plus grandes que la largeur ou hauteur maximale" sur "Oui", tous les fichiers téléchargés seront redimensionnés en fonction de ce que vous aurez défini dans "Largeur ou haiteur maximale pour les images/vidéons téléchargées (pixels)"
  • "Redimensionner automatiquement les images qui sont plus grandes que la largeur ou hauteur maximale" à "Oui: Tout le monde"
    Comme expliqué plus haut, vous devez définir ce paramètre à "Oui" pour être sur que les visiteurs n'aient pas de message d'erreur si ils téléchargent des fichiers plus grands que ce qu'ils devraient être. Coppermine redimensionnera les fichiers sur le serveur si nécessaire

Les paramètres ci-dessus sont les plus importants si vous décidez de désactiver la fonction d'affichage des images de grande taille dans une fenêtre pop-up, puisqu'ils économisent le maximum de place sur le serveur et rendent impossible pour les visiteurs de voir les images dans un plus grande résolution.

Autoriser les images originales dans une fenêtre pop-up en gagant un maximum de place

"Créer des images intermédiares " sur "Non" dans la configuration ne créera pas une galerie ou vous n'aurez pas d'images intermédiaires inclues dans l'affichage; à la place, les images originales seront affichées et redimensionnées grace à un mechanisme HTML. Cette option est recommandée si la majorité de vos images doivent être vues en grande taille: de cette manière, le navigateur mettra les images en cache et les fenêtres pop-up d'affichae des images de grande taille s'afficheront très rapidement.

Le fait de ne pas vrée d'images intermédiares aura bien entendu un impact dur l'espace web éconnomisé: ne pas créer d'images écocomisera bien entendu un peu d'espace. Toutefois, si vous avez un problème de place sur votre serveur, vous feriez mieux de limiter la "largeur ou haiteur maximale des images/vidéos téléchargées" et d'activer le redimensionnement d'images à ces dimensions (option "Redimensionner automatiquement les images qui sont plus grandes que la largeur ou hauteur maximale") - les appareils photos numériques actuels font des images énormes, avec des résolutions telles qu'elles utilisent beaucoup de place de stockage. Le redimensionnement de ces images à des résolutions qui ont un sens au niveau du web est tou à fait adapté pour toutes les sortes de galeries.

Exemple de configuration:
  • Paramètre de configuration "Créer des images intermédiares" → "Non"
  • Paramètre de configuration "Largeur ou hauteur maximale pour les images/vidéos intermédiaires" → "600"
  • Paramètre de configuration "Largeur ou hauteur maximale pour les images/vidéos téléchargées" → "1280"
  • Paramètre de configuration "Redimensionner automatiquement les images qui sont plus grandes que la largeur ou hauteur maximale" → "Oui: Tout le monde"

Paramètrage "standard" avec utilisation modérée d'espace par les fichiers

Si vous avez assez de place sur votre serveur et de bande passante et que vos visiteurs ont une bande passante décente, il est recommandé d'autoriser la création d'images intermédiaires et d'autoriser l'affichage des images de garnde taille dans des fenêtres pop-up pour les visiteurs qui voudront les voir en détail.

Exemple de configuration:
  • Paramètre de configuration "Créer des images intermédiares" → "Oui"
  • Paramètre de configuration "Largeur ou hauteur maximale pour les images/vidéos intermédiaires" → "800"
  • Paramètre de configuration "Largeur ou hauteur maximale pour les images/vidéos téléchargées" → "1600"
  • Paramètre de configuration "Redimensionner automatiquement les images qui sont plus grandes que la largeur ou hauteur maximale" → "Oui: Tout le monde"

Votre compte administrateur

Il est impératif de configurer correctement votre compte administrateur et de mémoriser les éléments de ce compte. Coppermine vous propose une option pour recréer un mot de passe si vous l'avez oublié, mais pour celà, il faut que vous ayez défini une adresse courriel pour chaque compte. Lorsque les utilisateurs s'enregistrent, ils doivent entrer une adresse courriel, il n'y a donc pas de problèmes pour eux. Toutefois, le compte administrateur que vous avez créer pendant l'installationde la galerie n'a pas encore d'adresse courriel. Vous devez allez à la page de votre profil en étant identifié comme administrateur et entrer une adresse courriel valide pour l'administrateur - de cette manière, vous pourrez demander la création d'un nouveau mot de passe si vous deviez oublier le votre par la suite.

Pour être absolument clair: il y a deux endroits dans Coppermine ou vous devez entrer une adresse courriel:

  • Le premier endroit est l'option de configuration "Adresse courriel de l'Administrateur ". Coppermine utilisera cette adresse comme adresse "de" lors de l'envoi de courriels (par exemple pendant l'enregistrement). Il est impératif de renseigner cette adresse courriel correctement. Si vous avez votre propre domaine, il peut être interressant d'avoir une adresse courriel en lien avec ce nom de domaine.
    Si votre nom de domaine est "exemple.com", vous devriez créer une adresse courriel nommée galerie@example.com et utiliser cette adresse dans la configuration de Coppermine. Coppermine ne crée pas cette adresse pour vous!
  • La seconde place est le profil de votre compte administrateur: vous devrez aussi définir une adresse courriel pour ce compte. Ce peut être n'importe quelle sorte d'adresse courriel - à confition que ce soit une adresse que vous suivez.

Ce n'est pas un problème si ces deux adresses courriel sont les mêmes: elles peuvent être identiques ou différentes. assurez vous simplement d'avoir défini les deux.

Vérification des téléchargements

Avant de promouvoir votre galerie Coppermine auprès du public, vous devrez vous assurer que le téléchargement fonctionne correctement, car ce sont les problèmes les plus courrants rencontrés par les utilisateurs, qui peuvent être causés par de multiples facteurs qu'il faut prendre en compte. Regardez la section de résoulution des problèmes de téléchargement si vous rencontrez des problèmes.

Penser à l'intégration (bridge)

Coppermine a été conçu comme une application autonome. Toutefois, beaucoup de personnes souhaitent l'intégrer à une autre application, donc, depuis la version cpg1.1.x, Coppermine propose un mechanisme qui permet aux utilisateurs d'intégrer Coppermine avec d'autres applications pour la gestion des utilisateurs. Le proncipal avantage est de ne demander qu'une seule identification à vos visiteurs pour l'ensemble du site (par exemple aussi bien pour votre galerie que pour votre forum), de ce fait ils n'ont pas à s'enregistrer deux fois ni à se rappeler deux différents identifiants.

Le bridge n'intègre pas visuellement Coppermine dans votre site (vous devrez créer un thème personnalisé pour faire cela).

Vous pouvez activer (ou désactiver) l'intégration à tout moment, mais vous devriez y réfléchir au moment de l'installation de Coppermine, parce qu'il y a certains problèmes à prendre en compte: si vous avez déjà des utilisateurs dans votre base de donnée Coppermine lorsque vous activez le bridge, la corrélation entre ces utilisateurs initiaux de Coppermine et les "nouveaux" utilisateurs de l'application avec laquelle vous bridger Coppermine sera perdue. La concéquence sera qu'il n'y aura plus de corrélation avec les actions que vos "anciens" utilisateurs avaient faites (téléchargement d'images, commentaires etc...) et les "nouveaux" utilisateurs de l'application avec laquelle vous intégrez Coppermine.

Pour éviter ces problèmes futurs, vous devez vous poser la question au moment de l'installation de Coppermine: voulez vous autoriser l'interraction des utilisateurs? avez vous l'intention de proposer un forum par la suite (ou toute autre application utilisant une gestion des utilisateurs)? Si votre réponse est "oui" ou "peut être", il est préférable d'activer le bridge avant de publier votre site et de laisser les visiteurs s'enregistrer.

Voyez les détails dans la section concernant l'intégration de la documentation.

Qu'est ce que vos visiteurs sont autorisés à faire?

Coppermine peut être utilisé pour beaucoup de choses: certains l'utilisent pour afficher leurs fichiers personnels et les rendre visibles à tous sur Internet, d'autres ne veulent uniquement qu'un nombre limité d'utilisateurs soient en mesure d'accéder à leur site uniquement pour une visualisation et sans interaction du tout. Une grande part de Webmasters veulent créer des sites communautaires, ou les utilisateurs peuvent participer en téléchargeant des fichiers, en commentant les fichiers des autres et en votant pour eux.

Vous pouvez utiliser Coppermine pour toutes ces utilisations (ou pour un mélange de celles ci), mais vous devrez être concient des possibilités et limitations d'abord:

Les utilisateurs "héritent" leurs autorisations du groupe auquel ils appartiennent. C'est pourquoi plusieurs paramétres de Coppermine sont définis par groupes. De cette manière, vous pouvez avoir plusieurs niveaux de permissions. Définissez les permissions par groupe en utilisant le gestionnaire de groupe

Changez l'aspect de votre galerie Coppermine

Vous pouvez modifier certains aspects de conception et de mise en page (par exemple affichage de vignettes au hasard sur la page d'index) dans la section correspondante de la configuration. Coppermine possède un moteur de thème vous permettant de personnaliser l'aspect de votre galerie. Vous pouvez choisir un des thèmes inclus dans Coppermine ou vous pouvez télécharger des thèmes provenant de contributions d'utilisateurs depuis la page de téléchargement du site de Coppermine. A partir d'un thème disponible, vous pouvez personnaliser votre propre thème pour faire en sorte que votre site ait un look unique et que la galerie ait le même aspect que le reste de votre site.

Réglages particuliers

Il y a beaucoup d'options de configuration à l'intérieur de Coppermine - utilisez les à bon escient. Comme beaucoup d'utilisateurs demandes des réglages particuliers (bien qu'il n'y ait pas de telle chose comme le réglage particulier qui corresponde à tout le monde), en voici juste quelques un:

Téléchargeur unique sans interaction d'utilisateur (galerie monolithique)

Seul un téléchargeur (vous l'administrateur) et peu pou pas d'interaction de l'utilisateur
Si vous êtes le seul à télécharger des images et que vous ne voulez pas ou peu d'interaction de la part des utilisateurs, désactivez tout ce que vous ne souhaitez pas permettre à vos visiteurs de faire. C'est généralement le cas si vosu êtes un artiste ou photographe: il n'y a pas de communauté crée avec la galerie - vous voulez juste montrer au public le contenu de votre site.

  • Désactivez l'enregistrement de nouveaux utilisateurs dans la configuration de Coppermine (Configuration → Enregistrement → Autoriser les nouvelles inscriptions). Cela va désactiver le lien "s'enregistrer" du menu. Les utilisateurs qui se rendraient manuellement sur la page d'enregistrement auront un message leur expliquant qu'ils ne peuvent pas faire cette opération
  • Désactivez la création de nouveaux albums pour les utilisateurs (Groupes → ligne "Registered" → Galerie personnelle → Autorisé = non) au cas ou vous n'aviez pas désactivé les enregistrements dès le début et qu'il reste des utilisateurs enregistrés.
  • N'autorisez pas les visiteurs à télécharger en allant dans le panneau de gestion des groupes et en paramétrant "Téléchargement dans les Albums Publics" à "non autorisé": Groupes → Ligne "Guests" → Téléchargement dans les Albums Publics → Autorisé = non. Si des visiteurs (ou un autre groupe d'utilisateur) n'est pas autorisé à avoir une galerie personnelle ni à télécharger dans les albums publics, le lien "Télécharger une image" du menu ne sera pas affiché.
  • Si vou sne voulez pas que vos visiteurs interagissent en notant, changez le réglage de note à non dans le paneau de gestion des groupes pour le groupe guests/anonymous: Groupes → Ligne "Guests" → Droits → Note = non. Si la notation est désactivée, le visiteur ne verra pas les étoiles de notation qui lui permetrait de noter l'image.
  • Si vous ne voulez pas quevos visiteurs postent des commentaires, mettez "non" pour les commentaires pour les visiteurs: Groupes → Ligne "Guests" → Droits → Commentaires = non.
  • Si vous ne voulez pas que vos visiteurs envoient des Ecartes, désactivez l'autorisation d'envoyer des Ecartes dans le panneau de gestion des groupes comme ci-après: Groupes → Ligne "Guests" → Droits→ Ecartes = non

      • Téléchargeur unique avec Interaction des utilisateurs

      Si vous devez être l'unique téléchargeur, mais que vous voulez créer une communauté autour devotre site avec des visiteurs enregistrésn qui postent des commentaires etc..., désactivez uniquement le téléchargement pour vos utilisateurs, mais activez toutes les autres fonctions que vous souhaitez autoriser:

      • Désactiver tous les téléchargements pour les visiteurs aussi bien que pour les utilisateurs enregistrés:
        • Groupes → Ligne "Guests" → Galerie personnelle → Autorisé = non
        • Groupes → Ligne "Registered" → Galerie personnelle → Autorisé = non
        • Groupes → Ligne "Guests" → Téléchargement dans les albums publics → Autorisé = non
        • Groupes → Ligne "Registered" → Téléchargement dans les albums publics → Autorisé = non
      • Pour éviter les spams dans les commentaires, vous devriez ne pas autoriser les commentaires pour les visiteurs, mais les uatoriser pour les utilisateurs enregistrés:
        • Groupes → Ligne "Guests" → Droits → Commentaires = non
        • Groupes → Ligne "Registered" → Droits → Commentaires = oui
        • Pour lutter contr eles spams dans les commentaires vous devriez activer l'approbation des commentaires et/ou Captcha pour les commentaires.
          Pour activer l'approbation des commentaires, allez dans la configuration de Coppermine et changez les réglages suivants: Configuration → Paramètres des Commentaires → Les Commentaires nécéssitent l'approbation de l'Administrateur .
          En plus (ou à la place), vous devriez activer captcha (une sorte d'image qui affiche des lettres alléatoires que le visiteur doit lire et entrer dans un champ afin de prouver qu'il est bien un humain et pas un fichu robot) pour les commentaires: Configuration → Paramètres de commentaires → Afficher Captcha (Confirmation visuelle) pour l'ajout de commentaires
      • Autoriser les visiteurs à noter votre contenu: vous activez la notation par niveau de groupe. Pour cela , allez dans le panneau de gestion des groupes et autorisez la notation pour tous les groupes possibles:
        • Groupes → Ligne "Guests" → Droits → Note = oui
        • Groupes → Ligne "Registered" → Droits → Note = oui
        Comprenez que ce n'est pas tout: pensez à ces réglages par groupes comme un commutateur général. Vous pouvez ajuster ces réglages individuellement pour chaque album: vous pouvez autoriser la notation pour un album et l'interdire pour un autre. Allez dans les Propriétés des albums de chaque album pour autoriser/interdire la notation pour cet album en particulier.
