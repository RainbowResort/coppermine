Problèmes connus

Les légères bizarreries et petits problèmes suivants sont connus et on peut espérer qu'ils seront corrigés dans les prochaines versions.

  • Les flêches utilisées pour la navigation d'une image à une autre s'affichent dans le mauvais ordre pour les langues au format rtl (lecture de droite à gauche)
  • La liste des mots clé renvoie un résultat incorrect pour les caractère asiatiques
  • Bug lié au navigateur: IE n'affiche pas correctement les largeurs de valeur 100%
  • La nouvelle interface de téléchargement par lots ne fonctionne pas avec toutes les configurations de serveurs. Les utilisateurs qui ont des problèmes avec cette fonction doivent utiliser l'interface classique à la place de la nouvelle.
  • Requête manquante dans la sortie de débuggage: La valeur de $CONFIG['debug_mode'] n'est pas fixée avant la première requête pour la table de configuration
  • Les onglets de page dans le Gestionnaire des utilisateurs ne se comportent pas de la même manière que les autres onglets
  • Le redimmentionnement des gif/png avec un fond transparent ne préservent pas les infos alpha
  • Carractères incorrects dans les courriels envoyés depuis Coppermine
  • Lat taille des vignettes supérieure à 120px cause un débordement du champs de texte au desus du cadre
  • Pas de vignettes d'albums pour les albums qui ne contiennent que des images liées par mots clés
  • Le recadrage et la rotation ne fonctionnent pas avec GD1
  • La rotation cause des distortions d'images pour les grands fichiers
  • la constrction<object><param></param></object> pour les fichiers multimédia n'est pas valide mais est necessaire pour une compatibilité ascendante
  • Les informations EXIF et IPTC sont perdues avec la rotation
  • Discrepancy between Coppermine internally using utf-8 and IPTC currently one supporting iso-8859-1. Subsequently, non latin characters like accented chars or Umlauts get distorted
  • Le groupe d'utilisateur "Banned" ne fonctionne pas comme souhaité
  • Les albums sont crées en utilisant XP publisher même en cas d'erreur de téléchargement
  • Pas de vignette de catégorie pour les galeries utilisateurs ni pour les catégories qui ne contiennet pas d'albums avec des images mais uniquement des sous-catégories
  • Film strip may change its height when navigating if thumb dimension is set to 'width'
  • Flash uploader (swfupload) won't work if the gallery is located in a password protected directory
  • Some rounded corner effects of the curve theme won't work across-browsers, but are browser-specific. They will not work in IE mainly. The browsers that can't display rounded corners will display "normal" corners.
  • Album moderator feature not clearly documented and therefore disabled (thread)
  • Category thumbnails appear larger than expected if thumb aspect is set to "exact" for the thumbnail setting "Use dimension" (thread)
  • If you allow guests to upload files to your gallery, they won't be able to add file details. This feature has been disabled as it's not possible to identify the user without fail (see thread). Additionally it may attract spammers to add textual content to your gallery. If you want to enable that feature anyway, find Coppermine's configuration table in the database and change the value of allow_guests_enter_file_details to '1'.
