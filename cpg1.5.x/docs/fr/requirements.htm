Qu'est ce qui est requis

Minimum requis

Il y a certains minimum requis qui doivent être mis en place avant de pouvoir installer Coppermine:

  • Un serveur Web

    Coppermine peut fonctionner avec tous serveurs web - Il y a des retours d'installations réalisées avec succès avec Apache (1.x supporté, 2.x recommandé), Microsoft Internet Information Server (IIS), Roxen WebServer, Abyss, JanaServer. L'équipe de développement de Coppermine recommande Apache 2.x.

  • Le support de PHP

    Le serveur Web doit comprendre le support de PHP (version 4.3.0 ou ultérieure), compilé avec le support de la librairie graphique GD ou les droits d'utilisation de la fonction exec() pour l'utilitaire "convert" d'ImageMagik capable de créer des vignettes et de redimmensionner les images. Le support de Mysql doit être aussi compilé avec PHP.
    Coppermine a été testé aussi bien avec PHP4 qu'avec PHP5. L'équipe de développementd e Coppermine encourage fortement l'utilisation de PHP5 (particulièrement depuis que le support d'aide de PHP4 a maintenant été arrêté).
    Il n'y a pas de support d'aide sur le forum de Coppermine pour les questions relatives au paramétrage des serveurs web et particulièrement pour ce qui est des questions sur php.ini ou sur la manière de paramètrer un serveur web vous même. Si vous avez des questions concernant le paramétrage d'un serveur web, demandez de l'aide à votre hébergeur.
    L'équipe de développement de Coppermine ne recommande ni n'apporte d'aide pour l'auto-hébergement!
    Nous ne pouvons pas vous dire ou se trouve votre fichier php.ini (vous pouvez chercher vous même en regardant la sortie du phpinfo) ni comment vous êtes autorisé à le modifier ou à l'écraser.
    Les raisons de ces règles strictes ont été discutées en détail, et si vous voulez savoir pourquoi elles existent, faites une recherche avec Google pour trouver ces sujets sur le forum. Fondamentalement, il est au-delà de la portée du support d'aide de Coppermine d'expliquer la configuration d'un serveur web.

    Les points suivant doivent être paramétrés correctement dans votre fichier php.ini. Nottez que le fichier php.ini est un fichier de configuration du serveur web et non une partie de Coppermine.

    • la directive file_uploads doit être On
    • upload_tmp_dir doit être paramétré avec un répertoire voalide
    • max_input_time fixé à une valeur d'au moins 30
    • post_max_size et upload_max_filesize fixés pour correspondre au plus grand fichier que vous souhaitez autoriser au téléchargement
    • memory_limit fixé à au moins 8M
      Vous aurez des problèmes de téléchargement si ceci est fixé à un trop petit nombre.
    • max_execution_time fixé à au moins 15
    • register_globals paramétré à Off
    Des détails peuvent être trouvés dans la section des restrictions côté serveur de la documentation.

  • Une base de donnée MySQL

    Vous avez besoin de MySQL version 3.23.23 ou plus récent. MySQL 4.1 est recommendé, MySQL 5 est supporté. L'utilisateur MySQL doit avoir les permissions pour créer (CREATE), modifier (ALTER), remplacer (REPLACE), délectionner (SELECT), mettre à jour (UPDATE), effacer (DELETE), insérer (INSERT) des données dans la base de donnée. La plupart des utilisateurs ont ces permissions avec les serveurs web. Si ce n'est pas le cas, vous ne pourez pas Coppermine ni aucun script. Si vous n'êtes pas sur d'avoir ces permissions, vérifiez avec votre hébergeur. La plupart des hébergeurs pourront certainement vous dire si Coppermine peut fonctionner sur leurs serveurs.
    Une base de donnée doit être crée afin que Coppermine puisse l'utiliser - le script d'installation ne crée pas de base de donnée pour vous, mais va créer automatiquement les tables et la structure des donnée dans votre base de donnée pendant l'installation. Dans la plupart des cas avec un hébergeur, vous aurez déjà une base de donnée créer par votre hébergeur. Sinon, il vous faudra la créer. Si vous devez créer une base de donnée, regardez avec votr ehébergeur quelle est la bonne methode pour le faire. Notez le nom de votre base de donnée, votre Nom d'utilisateur et le mots de passe. Il vous faudra les trois pour installer correctement Coppermine.
    Synthèse des Exigences de base de données:
    • MySQL 3.23.23 ou plus récent
    • privilèges pour utiliser les commandes suivantes:
      • CREATE
      • ALTER
      • REPLACE
      • SELECT
      • UPDATE
      • DELETE
      • INSERT
    • Un minimum de 500 kilobytes d'espace stockage dans la base de donnée, bien que plus soit hautement recommandé

  • Une librairie graphique:

    Coppermine repose sur la présence d'une librairie graphique pour créer des copies redimensionnées des images que vous téléchargez. Actuellement, Coppermine peut utiliser soit GD version 1.x ou 2.x (PHP doit être compilé avec) ou ImageMagick.
    (regardez Qu'est ce q'ImageMagik et Qu'est ce que GD)

C'est certainement une bonne idée de contacter en premier votre hébergeur pour savoir si ils sont au courrant de problèmes avec l'installation de Coppermine sur leurs serveurs.


Donc Coppermine peut fonctionner sur tous les seveurs web qui ont les minimums requis mentionnés plus haut, L'équipe de développement de Coppermine ne peut pas vous aider pour configurer votre serveur. Comme indiqué, il y a un minimum requis à vérifier et à mettre en place avant d'installer Coppermine. Notez que l'équipe de Coppermine ne recommande pas l'hébergement personnel sauf si vous avez une bonne expérience dans l'administration de serveurs web et que vous savez comment faire.

Requis personnels

Coppermine est destiné aux webmasters qui ont au moins un savoir faire de base en HTML et une certaine expérience dans la mise en place de scripts. L'équipe de dévelopement de Coppermine sait que certains utilisateur qui ont peu ou pas d'expérience dans ce domaine utilisent les auto-installeurs pour configurer leur galerie, ce qui est très bien. Nous vous demandons simplement de lire attentivement cette documentation avant de poser des questions sur le forum d'aide de Coppermine, la plupart des réponses aux questions de débutants s'y trouvent. Vous pouvez utiliser Coppermine sans problèmes avec les themes qui se trouvent dans le pack. Néanmoins, si vous voulez modifierun theme existant ou crére votr epropre theme (qui correspond à l'aspect du freste de votre site), vous devrez avoir des connaissance de base voir avancées en HTML et CSS . Vous n'avez pas obligatoirement besoin de connaissances en programation PHP pour utiliser ou administrer uen galerie Coppermine.
