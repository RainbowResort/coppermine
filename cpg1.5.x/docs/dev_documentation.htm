Editing the documentation

Target audience

This part of the documentation is not meant for end users of Coppermine, but only for developers. There is no support for this section, it comes as-is.

Edit existing documents

To edit existing documents, simply use your favorite plan text editor - it is all plain HTML plus the JavaScript navigation

Add new documents

To add new documents, follow the steps below:

  • just use an existing document as template to copy from. Save it within the docs folder (giving it a unique, web-save name that is meaningfull.
  • Edit out the content of the new document after the header section, i.e. between 
    <body>
<script type="text/javascript">
  cpgDocHeader();
</script>
<div id="toc">
<script type="text/javascript">
  cpgDocToc();
</script>
<noscript>
<a href="toc.htm">Table of Contents</a>
</noscript>
</div>
    and 
    <script type="text/javascript">
  cpgDocFooter();
</script>

</body>
</html>
    This is the section where your actual content should reside
  • Edit the <title>-tag to reflect your document's content
  • Edit script.js to add your document to the JavaScript navigation:
    Find d = new dTree('d'); and edit the section below it. The syntax is d.add(UniquePageID,ParentMenuItemID,'Navigation title','FileName'); You'll notice that the PageIDs don't have to increment by one - they can increment by any number. That's why I initially made them increase by 10, so you can easily add your new line (that corresponds to your new document) exactly at the place where it is suppossed to show up within the navigation without the need to review the PageID numbering scheme of the documents beneath your entry.
    (Review the dtree documentation for details)
  • Edit toc.htm and add a link to the plain HTML table of contents. This is necessary to allow search engine spiders to crawl the documentation, so please do not skip this step. The structure should be pretty self-explanatory (as suggested: plain HTML).
  • After taking care of creating the file and displaying it in the navigation, it's time to come up with your content. Make sure to use meaningfull headlines (preserving the existing <h1>, <h2>, <h3> ... structure). Preferably add anchors as well, making an icon () appear next to each headline that allows easy linking (this should be particularly helpfull on the support board - you can add references to particular documentation sections in no time). Take a look at the existing documents for an example
  • Finally, save your changes and add/commit your file(s) to the subversion repository.
